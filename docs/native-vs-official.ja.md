# mureo標準ツール と 公式MCP の違い（Google / Meta）

> 状態: 2026-06-16。各広告プラットフォームの公式MCPはベータであり、提供ツール
> は時期によって変化します。特定機能に依存する前に最新状態を再確認してください。
> 関連: [architecture.md](./architecture.md)、[byod.ja.md](./byod.ja.md)。

## 概要

mureo は自前の **標準（native）** ツール群（Google Ads、Meta Ads、GA4、Search
Console、Yahoo、LINE、Logly）を提供します。一方で各広告プラットフォームは
**公式MCPサーバー** を公開しており、mureo はそれらを *ドライバ* として
インストール・登録できます。両者は排他ではありません。mureo がコントロール
プレーン（司令塔）であり、公式MCP はプラットフォーム自身のAPIへの薄い
ドライバです。

Google と Meta における実務上の違い:

| | Google Ads | Meta Ads |
|---|---|---|
| 公式MCP | `googleads/google-ads-mcp`（pipx, ADC） | ホスト型 `https://mcp.facebook.com/ads`（OAuth） |
| 公式ツール数 | **3 — 読み取り専用** | **29 — 読み取り＋書き込み** |
| 公式で変更操作できる？ | 不可 | 可（本番アカウントへ即時反映。取り消し/下書き/確認なし） |
| mureo標準ツール数 | **82** | **88** |
| mureo標準で変更操作できる？ | 可 | 可 |
| 安全機構（ロールバック / 操作ログ / 戦略ゲート） | 標準のみ | 標準のみ |
| BYOD / `mureo demo`（CSV、ライブAPI不要） | 標準のみ | 標準のみ |

## Google Ads

### 公式MCP — `google-ads-official`

- パッケージ `googleads/google-ads-mcp` を pipx で導入。**ADC** 認証
  （`GOOGLE_ADS_DEVELOPER_TOKEN` + `GOOGLE_APPLICATION_CREDENTIALS`。MCC
  アクセス時は任意で `GOOGLE_ADS_LOGIN_CUSTOMER_ID`）。
- 公開ツールは **3つ、すべて読み取り専用**:
  - `search` — 読み取り専用の GAQL クエリ実行
  - `get_resource_metadata` — リソース型のスキーマ/メタデータ取得
  - `list_accessible_customers` — アクセス可能なアカウントID一覧
- **作成・変更は一切不可。** キャンペーン / 広告グループ / 広告 / キーワード /
  予算の作成・編集、ステータス変更、推奨の適用、コンバージョン設定は
  できません。

### mureo標準 — Google Ads（82ツール）

読み取りに加えて **書き込み** も可能。タスク単位の「意図を持った」ツールとして
提供されます:

- キャンペーン、広告グループ、広告（ディスプレイ + 画像アップロード含む）
- キーワード / 除外キーワード（追加 / 一時停止 / 監査 / 提案 / 重複排除）
- 予算（取得 / 更新 / 作成 / 再配分 / 効率分析）
- コンバージョン（一覧 / 作成 / 更新 / タグ / パフォーマンス）
- 検索語句（レポート / 分析 / レビュー）、サイトリンク、コールアウト
- 推奨（一覧 / 適用）、オークション分析、入札調整
- デバイス / 地域 / 時間帯ターゲティング、変更履歴
- パフォーマンス分析、コスト増加調査、ヘルスチェック、RSAアセット監査
- BtoB最適化、ランディングページ / クリエイティブ調査
- モニタリング目標（配信 / CPA / CV / コンバージョン0検知）、スクリーンショット取得
- 横断的なコントロールプレーン: ロールバック計画/適用、操作ログ、
  STRATEGY/STATE 連携、学習インサイト

### Google — それぞれで何ができるか

| 機能 | 公式MCP | mureo標準 |
|---|:---:|:---:|
| パフォーマンスデータの読み取り | 可 | 可 |
| 任意の生GAQLクエリ | 可 | 一部（専用レポートツール経由。自由記述GAQLは不可） |
| キャンペーン / 広告グループ / 広告の作成・更新 | 不可 | 可 |
| キーワード / 除外キーワード管理 | 不可 | 可 |
| 予算の作成 / 更新 / 再配分 | 不可 | 可 |
| コンバージョン設定・タグ付け | 不可 | 可 |
| 推奨の適用 | 不可 | 可 |
| 診断 / ヘルスチェック / モニタリング | 不可 | 可 |
| ロールバック / 監査ログ / 戦略ゲート | 不可 | 可 |

**結論（Google）:** 公式MCPは読み取り専用のレポート窓口（自前でGAQLを書きたい
場合に有用）。あらゆる *操作* と安全機構は mureo標準が担います。

## Meta Ads

### 公式MCP — `meta-ads-official`

- ホスト型 `https://mcp.facebook.com/ads`（Meta Ads AI Connectors、2026-04-29
  にオープンベータ）。**claude.ai のコネクタ** としてブラウザOAuth（Meta
  Business ログイン）で登録します。動的クライアント登録に非対応のため、
  Claude Code のユーザースコープMCPとしては追加できません。
- **29ツール、読み取り＋書き込み**。5グループ構成:
  - キャンペーン管理（5）: キャンペーン / 広告セット / 広告の作成、エンティティ更新、有効化
  - コマース向け商品カタログ（10）: カタログ、フィード、商品セット、商品、フィード品質診断
  - アカウント / ページ / アセット（3）: アカウント / エンティティ / ページの参照
  - データセット / トラッキング（4）: Pixel・コンバージョンAPIの **診断**（イベント品質、統計、エラー）— 読み取りのみ
  - インサイト / パフォーマンス（7）: レポート、ベンチマーク、異常検知、トレンド、機会スコアリング
- **書き込みは本番アカウントへ即時反映。取り消し・下書き・確認画面なし。**
  Meta自身の推奨緩和策: 最初の数日は読み取り専用権限にする、または
  アカウントレベルの予算上限を設定する。
- **公式Metaが「できない」と明示される領域:** オーディエンス / カスタム /
  類似オーディエンスの作成、コンバージョンAPIイベントの **送信**（診断のみ）、
  リードフォームの作成・リード取得、A/Bテスト設定、自動化ルール、
  クリエイティブ生成（コピー、画像、カルーセル、コレクション、ダイナミック）、
  Instagram固有の操作。

### mureo標準 — Meta Ads（88ツール）

公式MCPと同等の操作領域に加え、上記の「できない」領域を **すべて** カバー:

- キャンペーン、広告セット、広告（CRUD + 一時停止 / 有効化）
- オーディエンス + **類似オーディエンス**
- クリエイティブ — 単体 / カルーセル / コレクション / ダイナミック / リード — + 画像・動画アップロード
- インサイト + ブレイクダウン、分析（パフォーマンス / オーディエンス / 配置 / コスト / 比較 / クリエイティブ提案）
- Pixel（一覧 / 取得 / 統計 / イベント）と **コンバージョンAPI送信**（購入 / リード）
- カタログ / 商品 / フィード
- **リードフォーム + リード取得 + CSVエクスポート**
- **スプリットテスト**、**自動化ルール**、投稿ブースト、**Instagram**（アカウント / メディア / ブースト）
- 横断的なコントロールプレーン: ロールバック、操作ログ、戦略ゲート、学習インサイト、BYOD

### Meta — それぞれで何ができるか

| 機能 | 公式MCP | mureo標準 |
|---|:---:|:---:|
| パフォーマンス / ベンチマーク / 異常検知の読み取り | 可 | 可 |
| キャンペーン / 広告セット / 広告の作成 | 可 | 可 |
| 予算 / ターゲティングの編集 | 可 | 可 |
| 商品カタログ / フィード（ショッピング） | 可（10ツール） | 可 |
| Pixel・CAPI のシグナル診断 | 可 | 一部（Pixel統計/イベント） |
| オーディエンス / 類似オーディエンス | 不可 | 可 |
| コンバージョンAPIイベントの **送信** | 不可（診断のみ） | 可 |
| リードフォーム / リード取得 / CSVエクスポート | 不可 | 可 |
| クリエイティブ生成（カルーセル / コレクション / ダイナミック / リード） | 不可 | 可 |
| スプリットテスト / 自動化ルール | 不可 | 可 |
| Instagram固有の操作 | 不可 | 可 |
| ロールバック / 監査ログ / 戦略ゲート | 不可 | 可 |
| 取り消し / 下書き / 確認のガードレール | 不可（なし） | 可 |

**結論（Meta）:** 公式MCPは本番のキャンペーンやカタログを動かせますが、
ガードレールが無く、いくつかの機能欠落（オーディエンス、クリエイティブ、
CAPI送信、リード、テスト、ルール）があります。mureo標準はこれらの欠落を埋め、
安全 / 監査 / 戦略のレイヤーを付与します。

## 共存のしくみ

mureo が標準で対応するプラットフォームの公式プロバイダをインストールすると、
mureo は自身のMCPサーバーブロックに `MUREO_DISABLE_<PLATFORM>=1` を設定し、
ツールの重複公開を防ぎます。issue #102 / PR #265 により、**公式プロバイダが
実際にクレデンシャル設定されるまで標準ツールは無効化されません** — 利用可能な
ツールが一つも無い状態に陥ることはありません。公式プロバイダを削除すると
フラグは解除され、標準ツールが再び有効になります。

- 標準 → 公式へ切替: `mureo providers add <provider>`（または configure ダッシュボードのトグル）
- 元に戻す: `mureo providers remove <provider>`
- Search Console には公式MCPが存在しないため、mureo標準が正典であり無効化されません。

## どちらを使うべきか

- **mureo標準を使う場面:** ロールバック/監査付きの書き込み操作、戦略駆動の
  自動化、CSVでのBYOD/デモ、オーディエンス / クリエイティブ / リード / テスト /
  ルール（Meta）、または Google Ads のあらゆる変更操作が必要なとき。
- **公式MCPを追加する場面:** プラットフォーム自身のファーストパーティな
  データ面（GoogleでのGAQL生クエリ）が欲しいとき、または Meta のホスト型
  コネクタ経由でライブ運用を動かしたいとき — 欠落とガードレールを補うため、
  mureo標準と併用するのが理想です。
